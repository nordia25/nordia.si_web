8) Motion System (GSAP) — final contract
8.1 DO / DON’T

DO

transform/opacity only

section-scoped init/cleanup

batch reveali

matchMedia (desktop/mobile)

reduced motion + lite mode

animacije šele po initial paint

DON’T

layout animation (width/height/top/left/margin/padding)

heavy filters/blend-mode med animacijo

off-screen animacije

scroll-jack

8.2 Motion tokens (konsistenca)

Micro: 0.15–0.25s

Reveal: 0.35–0.7s

Hero: 0.8–1.2s (redko)

8.3 Hero “Nordia Lightfield” (svetloba je ključ)

Cilj: specular highlight se premika po “materialu” hero fotografije (Apple/Linear vibe), ne “2015 parallax”.

Desktop behavior:

pointer-driven specular highlight (damping, “težka luč”)

minimal depth shift 10–20px (sekundarno)

Mobile behavior:

brez miške: subtilen “light sweep” loop 6–10s

tilt samo, če je brez UX frikcije in 100% smooth (nice-to-have)

8.4 Kill-switch (nedvoumen)

Lightfield se degradira, če velja katerikoli pogoj:

Hard triggers (takoj):

prefers-reduced-motion: reduce → Lightfield OFF

save-data ON → lite mode

deviceMemory <= 4 ali hardwareConcurrency <= 4 → lite mode (Lightfield OFF)

Runtime triggers (med uporabo, ko je hero v view):

FPS < 30 za ≥ 500ms

ali 2× zapored “long frame” > 50ms med pointer/scroll interakcijo

Degradacija (final):

izklopi specular highlight + depth shift

pusti statično hero sliko

ohrani samo minimal fade-in tipografije (transform/opacity)

Fallback UX (brez messaginga):

user ne sme videti “disabled”; fallback je namerno lep.
